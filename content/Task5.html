請整理 https://cyberbotics.com/doc/guide/introduction-to-webots 中重要的英文單字與用法, 並且針對其中所使用的英文文法逐句說明:

重要單字及用法解析：

Simulation - 模擬 Webots 用於創建機器人模擬的環境，這裡指的是模擬機器人的行為和環境反應。

Rapid prototyping - 快速原型設計 這是一種開發過程，用來快速創建和測試產品原型。

Locomotion schemes - 移動方式 指的是機器人移動的方式或模式，如輪式、腿部式或飛行式。

Actuators - 執行器 是使機器人運作的裝置，如馬達、車輪等。

Controller - 控制器 在 Webots 中，控制器是負責控制機器人行為的程式。

Supervisor Controller - 監控控制器 這種控制器可以執行人類操作員通常才會執行的任務，並控制整個模擬環境。

Physics plugin - 物理插件 用來修改 Webots 模擬中的物理行為。

文法分析：

"Webots is a professional mobile robot simulation software package."

Webots (主詞) + is (連繫動詞) + a professional mobile robot simulation software package (主詞補語)。 這句是簡單的描述句，告訴我們 Webots 是一款專業的移動機器人模擬軟體。

"It offers a rapid prototyping environment, that allows the user to create 3D virtual worlds with physics properties such as mass, joints, friction coefficients, etc."

這句中使用了定語從句 "that allows the user to create..."，修飾前面的環境。 "that allows": 用來描述這個環境具備的功能，這是一個限定性定語從句。

"The user can add simple passive objects or active objects called mobile robots."

"called mobile robots" 是分詞短語，修飾 "active objects"。

這裡的 "called" 用來說明 "active objects" 的名稱是 "mobile robots"。

"Webots also contains a number of interfaces to real mobile robots, so that once your simulated robot behaves as expected, you can transfer its control program to a real robot like e-puck, DARwIn-OP, Nao, etc." "so that"

這個引導的目的從句解釋了為什麼 Webots 提供這些接口（目的是為了當模擬機器人按預期運行時，可以將控制程式轉移到真實機器人）。

"A world, in Webots, is a 3D description of the properties of robots and of their environment."

這是定義句，用來解釋 "world" 是什麼。

句中的 "in Webots" 用來說明這個概念是在 Webots 中的特定含義。

"A controller is a computer program that controls a robot specified in a world file." 這是一個定義句，說明了什麼是 "controller"。

其中 "that controls" 是定語從句，修飾 "controller"。

常見的英文文法結構 定義句：通常用來解釋某個名詞的含義。

例子："A world is a 3D description of the properties of robots."

定語從句：用來修飾名詞，提供更多描述。

例子："A controller is a program that controls a robot."

目的從句：用來說明某一行動的目的。

例子："so that you can transfer its control program to a real robot."

條件句：通常描述在某些條件下會發生的情況。

例子："once your simulated robot behaves as expected, you can transfer its control program..."

這篇資料簡單而清楚地介紹了 Webots 的基本功能及其應用，並且在文法上多使用定義性和解釋性的句型，幫助讀者理解軟體的各個方面。

Webots 使用者指南 R2025a 摘要：

Webots 介紹 什麼是 Webots？ Webots 是一款專業的移動機器人模擬軟體。

它允許使用者創建具有物理屬性的 3D 虛擬世界，如質量、關節和摩擦係數等。

使用者可以添加被動物件或稱為移動機器人的主動物件，這些機器人可以具有不同的移動方式，如輪式、腿式或飛行機器人。

這些機器人可裝配多種感測器和執行器（例如：相機、馬達、距離感測器等）。

Webots 允許使用者為每個機器人編程，實現所需的行為。

它還具有與實際機器人（如 e-puck、Nao 和 DARwIn-OP 等）接口的功能。

我可以用 Webots 做什麼？ Webots 非常適合用於與移動機器人相關的研究和教育項目，涵蓋的領域包括：

移動機器人原型設計（學術研究、汽車產業、航空業等）

機器人運動研究（如人形機器人、四足機器人等） 多代理研究（如群體智慧、協作型移動機器人等）

自適應行為研究（如遺傳算法、神經網絡、人工智慧等）

機器人教學（機器人學、C/C++/Java/Python 程式語言教學等）

機器人比賽（例如 Robotstadium 或 Rat's Life）

我需要知道什麼才能使用 Webots？ 若要開發模擬，您需要具備基本的程式設計知識，特別是 C、C++、Java、Python 或 MATLAB 這些語言，用於編寫機器人控制程式。

如果您不熟悉這些語言，也可以使用簡單的圖形化程式語言 BotStudio 來編程 e-puck 和 Hemisson 機器人。

若您希望創建自定義的機器人模型或在模擬環境中加入特殊物件，則需要具備基本的 3D 電腦圖形和 VRML97 描述語言知識。

如何獲得使用者支援？ Cyberbotics 提供付費的使用者支援服務，包括 Premier Service 和定制開發服務。

社群支援也可通過以下平台獲得： Robotics StackExchange（用於詢問 Webots 相關技術問題）

GitHub Discussions（用於與 Webots 開發和文檔相關的問題）

GitHub Issues（用於報告 Webots 中的錯誤）

Discord（用於快速問題解答和社群討論）

Webots 模擬組成部分： Webots 模擬包括以下組件： World 文件 (.wbt)： 定義一個或多個機器人及其環境，並可能依賴外部的 PROTO 文件 (.proto) 和紋理。

控制程式： 用於控制上述機器人的程式，可使用 C、C++、Java、Python 或 MATLAB 編寫。

物理插件（可選）： 可用於修改 Webots 正常的物理行為。

什麼是 World？ 在 Webots 中，世界是對機器人及其環境屬性的 3D 描述。

它包含每個物件的位置、方向、幾何形狀、外觀（如顏色或亮度）、物理屬性、物件類型等的描述。

世界是以階層結構組織的，物件可以包含其他物件（例如：一個機器人可以包含兩個車輪、一個距離感測器以及一個包含相機的關節）。

世界文件不包含機器人的控制程式代碼，它僅指定每個機器人所需的控制程式名稱。

世界文件儲存在 Webots 專案中的 "worlds" 子目錄。

什麼是 Controller？ 控制器是控制在世界文件中指定的機器人的程式。

控制器可以用 Webots 支援的任何程式語言編寫：C、C++、Java、Python 或 MATLAB。當模擬啟動時，Webots 會啟動指定的控制程式，每個控制程式作為單獨的進程運行，並將控制程式與模擬中的機器人進行關聯。

注意，儘管多個機器人可以使用相同的控制程式代碼，但每個機器人會啟動一個獨立的進程。

什麼是 Supervisor Controller？ Supervisor 控制器是機器人控制器的一種，其 supervisor 欄位設置為 TRUE。它可以執行通常只能由人工操作員完成的操作，而不是由實際機器人完成。

Supervisor 控制器可以用上述任一程式語言編寫，但與普通的機器人控制器不同，它將具有訪問特權操作的權限。特權操作包括模擬控制，例如將機器人移動到隨機位置、捕捉模擬畫面等。 這篇指南簡要概述了 Webots 的功能、使用方法以及如何開始進行模擬和機器人編程。

請詳細說明如何採手動方式在 Webots 建立下列場景:

開啟 Webots 軟體

首先，啟動 Webots 軟體。如果你尚未安裝 Webots，可以從其官方網站下載並安裝。

創建新專案

在 Webots 中，點選 "File"（檔案） > "New World"（新世界）來創建一個新的世界。 這會自動打開一個新的場景並顯示在視窗中。

設置地面 (Ground)

Webots 中的地面通常是一個 Plane 物體，並且預設會包含在新創建的場景中。如果你沒有看到地面，或者想要自定義它： 在左邊的 Scene Tree（場景樹）中，右鍵點擊 WorldInfo > Add > Solid。 在 "Solid" 下，選擇 Plane，這將自動添加一個平面。 點擊平面並調整其屬性： Translation：調整平面的位置，通常設置為 (0, 0, 0)，表示它位於世界的中心。 Size：設定為 (100, 0.1, 100) 使它成為一個大型的地面。

加入機器人 (例如 e-puck)

在 Webots 中，你可以從庫中選擇機器人模型來添加： 在 Scene Tree 中右鍵點擊 WorldInfo，選擇 Add > Robot。 點擊 Add 後，選擇 e-puck 機器人（它是 Webots 的預設機器人之一）。 e-puck 會自動添加到你的場景中。調整機器人的位置： Translation：設定為 (0, 0, 0) 或任何你希望它出現的位置。 在機器人的屬性中，調整其控制程式（如果需要），或者保留預設。

加入障礙物 (Obstacle)

假設你想添加幾個簡單的障礙物，這裡是加入立方體的步驟： 右鍵點擊 WorldInfo，選擇 Add > Solid。 在 "Solid" 下，選擇 Box，這將添加一個立方體。 調整立方體的位置和大小： Translation：設定為 (5, 0.5, 0)，將立方體放在地面上方。 Size：設定為 (1, 1, 1)，控制障礙物的大小。 重複這個過程來添加更多的障礙物。你可以調整位置來創建更具挑戰性的場景。

設置場景屬性

WorldInfo：可以在 WorldInfo 節點下修改世界的基本信息，比如 name（名稱）或 simulation time step（模擬時間步長）。 調整 time step（模擬步長），這可以影響模擬的精度。通常設置為 32 毫秒（0.032）是合理的。

檢查並儲存場景

完成以上步驟後，你可以按下 Run 按鈕來檢查場景。 如果你滿意這個場景，可以選擇 File > Save As 來保存你的場景，並給它一個有意義的名稱。

編寫控制程式

如果你希望機器人能夠根據特定邏輯運行（例如避開障礙物），你可以編寫控制程式： 在 Scene Tree 中，找到 e-puck 機器人，然後右鍵選擇 Add > Controller。 選擇一種編程語言（例如 C、C++、Python 等），並開始編寫控制程式來定義機器人的行為。

運行模擬

完成場景的設置後，你可以點選上方的 Play 按鈕來運行模擬。 這樣你就可以看到你的機器人如何與障礙物互動，並且可以進行調整和優化。

進一步的設置（可選）

你可以添加更多物體，如樹木、建築、燈光等，以使場景更加豐富。 若你希望進行更高級的控制，還可以編寫 Supervisor 控制器來進行全局管理和控制。 結論 通過以上步驟，你就能夠在 Webots 中手動創建一個簡單的模擬場景，包括地面、機器人和障礙物。你可以繼續探索 Webots 提供的其他功能，像是設置傳感器、進行機器人控制程式編寫，以及進行更多模擬測試。

設置機器人的控制程式

在畫面上方的工具列中，點選 Play（播放）按鈕來啟動模擬。 這時候，你會看到 e-puck 會在平面上運行（如果你寫了控制程式，機器人會根據程式執行對應的行為）。

進一步調整與測試

如果機器人或障礙物的位置不符合預期，你可以回到 Scene Tree 中直接選擇對應的物體，並調整它們的位置和屬性。 如果需要加入更多功能，例如傳感器或更複雜的障礙物，則可以通過右鍵點擊 Solid，選擇 Add 來選擇不同類型的物體，例如 Sensor、Camera 等。 特別注意的要點： 物體屬性設置：每個物體（如機器人、障礙物等）都有許多屬性（例如位置、大小、顏色等），需要確保你調整好這些屬性，讓物體符合你的需求。 控制程式的選擇：如果你選擇使用自定義控制程式，記得指定正確的控制程式名稱，並確保該控制程式已經存在於正確的資料夾中。 時間步長 (Time Step)：模擬的時間步長影響物理計算的精度和運行速度。步長過短可能使模擬變得很慢，過長則可能使物理模擬不夠精確。

場景的儲存：在進行修改時，記得定期保存場景，避免因為程式崩潰或其他問題而丟失資料。

3D 模型的載入：如果需要載入外部 3D 模型或特殊物體，Webots 支援使用 PROTO 和 VRML97 格式來導入自定義模型。

結論： 通過這些步驟，你可以使用 Webots 的圖形介面手動創建一個簡單的場景，並讓機器人在其中運行。利用圖形介面可以簡化許多設定過程，但同時也需要熟悉場景結構與控制程式設置。
